**Paper:** π³: Permutation-Equivariant Visual Geometry Learning, ICLR 2026, pi3
**Summary:** π³ is a fully feed-forward visual geometry model that accepts an arbitrary number of RGB images (single images, video sequences, or unordered multi-view sets) and predicts, per image, a dense pixel-aligned local 3D point map, an affine-invariant camera pose, and a per-pixel confidence map — all in a single forward pass with no per-scene optimization. Its central contribution is a permutation-equivariant architecture that eliminates the conventional fixed reference view: point maps are defined in each frame's own camera coordinate system and camera poses are supervised via relative-pose losses, making the system robust to input ordering and reference-frame choice. Reconstruction is up to an arbitrary similarity transformation, not metric scale.

1. **3D reconstruction:** Yes. The model outputs pixel-aligned 3D point maps X_i ∈ R^{H×W×3} for each input image, one 3D point per pixel; multi-view point map quality is evaluated on 7-Scenes, NRGBD, DTU, and ETH3D (Sec. 3.1, eq. 1, p. 4; Sec. 4.2, Tables 2–3, p. 7).
2. **Dense reconstruction:** Yes. The geometry output is a fully dense per-pixel point map; the "sparse/dense" column labels in Table 2 refer to the density of input view sampling, not to sparse geometry output (Sec. 3.1, p. 4; Sec. 4.2, p. 7).
3. **Reference frame:** Egocentric. Each per-frame pointmap is defined in that frame's own camera coordinate system; the model's permutation-equivariant design deliberately eliminates any canonical reference frame. Camera poses are supervised via relative-pose losses, and the output coordinate system is arbitrary up to a global similarity transform — Umeyama+ICP alignment is required to compare with ground truth (Sec. 3.1, p. 4; Sec. 3.3, pp. 5–6).
4. **Metric scale:** No. The network predicts point maps up to an unknown but consistent scale factor across the N input images; the optimal scale s* is solved only during training as part of the loss. Predicted camera trajectories and point maps are evaluated after scale or Sim(3) alignment, so metric scale is not recovered (Sec. 3.2, eqs. 4–5, p. 5; Sec. 3.3, pp. 5–6; Sec. 4.2, p. 7).
5. **Point tracking:** No. The model output tuple contains only per-image poses, local point maps, and confidence maps. No cross-frame point correspondences, track identities, or matching outputs are defined or evaluated (Sec. 3.1, eq. 1, p. 4; Sec. 4, pp. 6–8).
6. **3D tracks:** No. π³ estimates per-frame geometry and camera poses; it does not output 3D point trajectories or temporally persistent 3D tracks (Sec. 3.1, p. 4; Sec. 4, pp. 6–8).
7. **2D tracks:** No. No 2D point tracking output or evaluation is described. If Q6 = Yes and Q12 = Yes, the auto-Yes rule does not apply (Sec. 4, pp. 6–8).
8. **Occlusion / visibility prediction:** No. The model predicts a confidence map C_i ∈ R^{H×W} per frame, supervised with a BCE loss where the target is 1 when the per-point reconstruction error falls below a threshold ε, else 0. This is a geometry reliability score, not an explicit occlusion or visibility flag derived from multi-view visibility reasoning (Sec. 3.1, p. 4; Sec. 3.2, p. 5).
9. **Multiple views:** Yes, arbitrary number. The model is defined for N input images where N is arbitrary; training batches contain 2 to 24 images and the architecture jointly processes all views via alternating view-wise and global self-attention. Inputs may be ordered (video) or unordered (image sets) (Sec. 3.1, p. 4; Appendix A.2, p. 13–14).
10. **Global context vs. pairwise vs. fixed-window:** Global. All N input views are processed jointly in a single forward pass through alternating view-wise and global self-attention layers; no pairwise decomposition or fixed temporal window is used (Sec. 3.1, pp. 4–5; Appendix A.1, p. 13).
11. **No GT extrinsics required at inference:** Yes. Ground-truth camera extrinsics are not provided as input at inference; the network predicts camera poses from raw RGB images, and camera pose estimation is evaluated as an output task (Sec. 3.1, p. 4; Sec. 4.1, pp. 6–7).
12. **Extrinsics as output:** Yes. Camera poses T_i ∈ SE(3) ⊂ R^{4×4} are explicit network outputs supervised via relative-pose rotation and translation losses, and evaluated on RealEstate10K, Co3Dv2, Sintel, TUM-dynamics, and ScanNet (Sec. 3.1, eq. 1, p. 4; Sec. 3.3, pp. 5–6; Sec. 4.1, Table 1, pp. 6–7).
13. **No GT intrinsics required at inference:** Yes. The input is raw RGB images only; no ground-truth camera intrinsics are required or listed as inputs at inference time (Sec. 3.1, p. 4; Appendix A.1, p. 13).
14. **Intrinsics as output:** No. The model's output tuple is (T_1, …, T_N), (X_1, …, X_N), (C_1, …, C_N) — camera extrinsics, local point maps, and confidence maps. No intrinsic calibration head or focal-length output is described anywhere in the paper (Sec. 3.1, eq. 1, p. 4; Appendix A.1, p. 13).
15. **Moving cameras:** Yes. Camera pose and trajectory estimation are central tasks, evaluated on moving-camera benchmarks including RealEstate10K, Sintel, TUM-dynamics, ScanNet, and Co3Dv2, as well as video depth estimation on Sintel, Bonn, and KITTI (Sec. 4.1, Table 1, pp. 6–7; Sec. 4.3, Table 4, p. 8).
16. **Dynamic content:** Yes. The paper explicitly states π³ can reconstruct both static and dynamic scenes. It trains on an internal dynamic scene dataset and evaluates on dynamic/video benchmarks (TUM-dynamics, Sintel with non-rigid content, Bonn). Figure 1 caption notes "both dynamic and static content" (Figure 1, p. 1; Introduction, p. 3; Sec. 3.4, p. 6; Sec. 4.1 and 4.3, pp. 6–8).
17. **Feed-forward inference:** Yes. π³ is described throughout as a feed-forward neural network performing a single forward pass with no per-scene optimization or test-time training. It achieves 57.4 FPS on KITTI on a single A800 GPU, versus DUSt3R's 1.25 FPS and VGGT's 43.2 FPS (Abstract, p. 1; Introduction, p. 3; Table 4, p. 8).

**Notes:** π³ is an RGB-only model trained end-to-end on 15 diverse datasets including synthetic (GTA-SfM, TartanAir, Habitat, Hypersim, BlendedMVG, MatrixCity, Mid-Air), real indoor (CO3D, WildRGB-D, ARKitScenes, ScanNet, ScanNet++), and an internal dynamic scene dataset (Sec. 3.4, p. 6). In the main experiments, the model is initialized from pretrained VGGT weights with the encoder frozen during training (Appendix A.2, pp. 13–14). The predicted camera poses are affine-invariant (up to similarity transformation), not metric; scale alignment is required for evaluation. The confidence maps are geometry reliability scores, not occlusion or visibility labels. Limitations noted by the authors include inability to handle transparent objects, less fine-grained detail than diffusion-based approaches, and grid-like artifacts from pixel-shuffle upsampling in high-uncertainty regions (Appendix A.8, p. 17).
