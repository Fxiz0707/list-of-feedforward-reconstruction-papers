**Paper:** Structure-from-Motion Revisited, CVPR 2016, COLMAP
**Summary:** COLMAP is an incremental Structure-from-Motion pipeline for unordered multi-view image collections. It extracts and matches local features, verifies correspondences geometrically, registers images via PnP, triangulates sparse 3D points from feature tracks, and refines all camera poses and points through iterative bundle adjustment. Its inference regime is a per-scene iterative optimization pipeline, not a feed-forward learned prediction pass.

1. **3D reconstruction:** Yes. The reconstruction stage explicitly outputs pose estimates for all registered images and a reconstructed set of 3D points X = {X_k ∈ R³} triangulated from feature tracks (Sec. 2.2, p. 4105).
2. **Dense reconstruction:** No. COLMAP reconstructs sparse triangulated feature-track points rather than a dense surface, dense depth map, or dense point cloud; experiments report sparse point counts and average track lengths (Secs. 2.1–2.2 and Table 1, pp. 4105, 4111).
3. **Reference frame:** Fixed world. All cameras and 3D points are expressed in a single SfM world frame established during global bundle adjustment; the gauge is fixed by the initialisation and not user-reassignable at inference time.
4. **Metric scale:** No. Monocular SfM from unordered uncalibrated images has an inherent scale ambiguity; COLMAP introduces no external metric reference (e.g., GPS, known baseline, or IMU), so absolute metric scale is not recovered. Evaluation uses reprojection error in pixels; the Quad dataset ground truth is used only for accuracy comparison, not as input (Secs. 2.2, 4.4, and 5, pp. 4105, 4108, 4110–4111).
5. **Point tracking:** Yes. Feature extraction and matching identify projections of the same scene points across images, and geometric verification constructs verified feature tracks used for triangulation (Secs. 2.1 and 4.3, pp. 4104–4105, 4107–4108).
6. **3D tracks:** No. Feature tracks are multi-view observations of static scene points; COLMAP does not output temporal 3D trajectories of moving points or dynamic objects (Secs. 2.1 and 4.3, pp. 4104–4105, 4107–4108).
7. **2D tracks:** Yes. The method forms sparse 2D feature tracks by concatenating geometrically verified correspondences that observe the same scene point across multiple images (Secs. 2.1 and 4.3, pp. 4104–4105, 4107–4108).
8. **Occlusion / visibility prediction:** No. COLMAP does not output per-point or per-track occlusion or visibility flags as a user-facing capability. Binary visibility vectors appear only internally in Sec. 4.5 to parameterize bundle adjustment grouping for efficiency, not as a predicted output (Sec. 4.5, p. 4109).
9. **Multiple views:** Yes. COLMAP's core input is an unordered collection of images from multiple viewpoints; it constructs a scene graph over all input images and estimates poses for all registered cameras (Secs. 2.1–2.2 and 5, pp. 4104–4105, 4110–4111).
10. **Global context vs. pairwise vs. fixed-window:** Global. Pairwise verified matches are aggregated into a scene graph; incremental reconstruction then builds one global model refined by local BA after each registration and global BA after sufficient model growth, so all registered images participate in optimization (Secs. 2.2 and 4.4, pp. 4105, 4108).
11. **No GT extrinsics required at inference:** Yes. Camera poses are estimated from scratch via PnP on 2D-3D correspondences and refined by bundle adjustment; no ground-truth extrinsics are provided as input (Secs. 2.2 and 4.4, pp. 4105, 4108).
12. **Extrinsics as output:** Yes. Camera poses P_c ∈ SE(3) for all registered images are primary outputs of the reconstruction stage, evaluated against ground-truth locations where available (Secs. 2.2 and 5, pp. 4105, 4110–4111).
13. **No GT intrinsics required at inference:** Yes. The method supports uncalibrated cameras and estimates intrinsic parameters (focal length, radial distortion) during registration and bundle adjustment via pure self-calibration (Secs. 2.2 and 4.4, pp. 4105, 4108).
14. **Intrinsics as output:** Yes. Focal length and radial distortion are estimated and refined per camera; the principal point is fixed at the image center, and degenerate intrinsic estimates are detected and filtered after global BA (Sec. 4.4, p. 4108).
15. **Moving cameras:** Yes. SfM is defined as reconstructing 3D structure from images taken from different viewpoints with moving cameras, and the pipeline explicitly handles moving-camera epipolar geometry and pose estimation (Secs. 1–2.2, pp. 4104–4105).
16. **Dynamic content:** No. The method targets static scene reconstruction and does not model independently moving objects or non-rigid scene content; observations with large reprojection errors (caused by, e.g., moving objects) are filtered as outliers (Secs. 3 and 4.4, pp. 4106, 4108).
17. **Feed-forward inference:** No. COLMAP is an iterative per-scene optimization pipeline involving feature matching, geometric verification, incremental image registration, triangulation, re-triangulation, filtering, and repeated local and global bundle adjustment; it is not a single feed-forward inference pass (Secs. 2.2 and 4.4, pp. 4105, 4108).

**Notes:** COLMAP is a classical, non-learned SfM system, not a neural reconstruction model. Input is an unordered image collection; experiments use RootSIFT features with vocabulary-tree approximate nearest-neighbor matching. The system can self-calibrate intrinsics only within assumed camera-model constraints (single radial distortion parameter for Internet photos). Outputs are sparse SfM cameras, feature tracks, and triangulated points — not dense geometry, not absolute metric-scale structure, and not dynamic 3D tracks. Scale ambiguity is a fundamental limitation of monocular SfM without an external reference. The open-source implementation is available at https://github.com/colmap/colmap.
